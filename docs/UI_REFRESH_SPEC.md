# UI Refresh Spec

## Visual Direction
- Tone: operational enterprise
- Surfaces: neutral light background with white working panels
- Color: reserved for active state and semantic status, not decoration
- Shape: `6-8px` radius for cards, buttons, inputs, pills
- Depth: light borders first, restrained shadows second

## Icon Rules
- Use one monochrome outline icon system across shell and module entry pages
- Use icons for top-level navigation, major actions, and meaningful status only
- Do not use emoji in navigation, cards, badges, or headings
- Keep icons small and structural; do not use oversized decorative hero icons

## Color Usage
- `--accent`: active navigation, focused controls, primary actions
- `--success`, `--warning`, `--danger`, `--info`: status only
- Module identity can appear as subtle labeling, not full-card gradients or bright strips

## Components
- Navigation: compact rows, quiet active state, outline icon + label
- Launch cards: small icon tile, strong title, short descriptive copy, single text action
- Buttons: primary solid, secondary neutral outline, ghost neutral
- Inputs: white field, visible border, blue focus ring
- System panels: white background, standard border, minimal shadow

## Content Rules
- Prefer direct operational labels over promotional phrasing
- Keep helper text short and factual
- Use lock/access states as neutral system messages
