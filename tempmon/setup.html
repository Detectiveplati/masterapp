📖 RD07 WiFi Gateway — Setup Guide

Step-by-step for TAG09 (temp, ch 3) and TAG08B (temp + humidity, ch 2)

🔌
Step 1 — Power on

Screw on the RF antenna before powering on.

Switch: Left = ON, Right = OFF

3 LEDs flash simultaneously = working.

LED guide:
🔵 Blue 0.1s flash = tag data received
🟢 Green 2s blink = forwarded to server
🔴 Red solid = power connected

💻
Step 2 — Connect TZConfig

1. Install PL-2303 RS232 driver (Win 10+)
2. Connect USB RS232 cable: small plug → gateway, large → PC
3. Device Manager → Ports → note COM port
4. TZConfig.exe → select COM port → Connect

In WiFi tab:
• Mode: WIFI
• SSID: kitchen WiFi name
• Password: WiFi password → Save

🌐
Step 3 — TCP/IP server address

In Data Transfer Protocol:
• Protocol: TCP/IP
• Transfer mode: TCP (not UDP)
• Server IP: copy from box at top ↑
• Port: copy from box at top ↑

• Interval: 10 seconds
• Click Save then Read to verify

Railway: Add TCP Proxy in Railway dashboard → Settings → Networking → Add TCP Proxy → port 4001

🏷️
Step 4 — Tag Settings ⚠️ Critical

In Tag Settings tab — add each sensor:

TAG09 → Channel 3
TAG08B → Channel 2

For each tag: Add → Type → Channel → 8-char TAG ID → Save

⚠️ Once any tag is added, gateway only forwards listed tags. Add all your sensors or they will be silently dropped.

🔘
Step 5 — Wake & verify

Press each tag button for ~1 second to force an immediate transmission.

🔵 Blue LED flash = gateway received the signal
🟢 Green blink = sent to server

Check the Live Scanner panel above. The tag should appear within a few seconds.

Default auto-interval: every 15 minutes.

✅
Step 6 — Register in app

Once a tag appears in Live Scanner or Detected Sensors:

1. Click Register next to it
2. Confirm model (TAG09 / TAG08B)
3. Choose equipment (Chiller / Freezer / Warmer)
4. Click Register Sensor

All future readings are stored automatically in the database.

🔋 Battery: ≥3.5V Good 🟢  |  3.3–3.5V Acceptable 🟡  |  <3.3V Replace now 🔴
📶 RSSI: 0 to −50 dBm Excellent  |  −51 to −80 dBm Good  |  below −80 dBm Weak — move gateway closer
ℹ️ TAG09 vs TAG08B — differences explained
TAG09 — Temperature only

• No humidity sensor (payload humi = FF = absent)
• Channel: 3 in TZConfig Tag Settings
• Good for: cold rooms, freezers, dry stores
TAG08B — Temp + Humidity

• Humidity resolution: 0.1% (2-byte field)
• Channel: 2 in TZConfig Tag Settings
• Good for: rooms where humidity matters

📖 How Temperature Thresholds Work

❄️ CRITICAL LOW
Below Critical Min
🟡 WARNING LOW
Min → Min + Buffer
✅ SAFE ZONE
Within limits
🟡 WARNING HIGH
Max − Buffer → Max
🔴 CRITICAL HIGH
Above Critical Max

Field meanings

Critical Min / MaxLegal/food-safety hard limits. Breach = critical alert.
Warning BufferDegrees inside the critical limit used for chart colouring only (yellow warning band). Does not trigger push notifications — only crossing Critical Min/Max does.
Target TempIdeal operating point. Shown on charts as a dotted line. Does not trigger alerts.
Alert Delay (min)Minutes out of range before an alert record is created. Use to suppress brief door-opening spikes.

🔥 Warmer-Specific: On / Off / Fault Detection

Because warmers are switched off during low-load periods, the system runs a state machine on each reading to tell the difference between a warmer that is off, warming up, active, or faulty. This means you won't get false alerts when a warmer is legitimately switched off.

⚠️ FAULT state: If the warmer is on (temp above room level) but hasn't reached the target (Critical Min) after the fault timeout, a FAULT push notification is sent. This catches warmers that are switched on but not heating effectively — e.g. element failure, door left open, or set to the wrong temperature.
Warmer config field Default Meaning
Room Temp Ceiling35°CAt or below this temp → warmer is considered off / cooling down.
Warmup Start Temp40°CAbove this temp → warmer is considered switched on and in the warming-up phase.
Off Confirm (min)20 minHow long to sit at room temp before confirming the state as OFF (not just a brief dip).
Fault Timeout (min)30 minIf the warmer has been on (warming up) for this long without reaching the target, it is flagged as FAULT and a push alert is sent. Most warmers reach temperature within 15–20 min.
Target (Critical Min)63°CThis is the temperature the warmer must reach to be considered ACTIVE. Use the Critical Min field on the unit — the state machine uses this as the target threshold.

💡 These values can be adjusted per unit by an administrator via the database if a specific warmer warms up slower or operates at a different typical room temperature.
